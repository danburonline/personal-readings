# AGENTS.md

You are operating in a personal research library of annotated scientific papers, essays, and technical documents.

## Language and Style

- Write in **British English** (e.g. "summarise", "behaviour", "organised")
- No em dashes. Use double hyphens (--) or restructure the sentence
- No AI-typical wording: avoid "delve", "crucial", "it's important to note", "landscape", "leverage", "streamline", "cutting-edge", "In conclusion", and similar filler
- No hedging or softening. State claims directly
- All output should read as if written by a researcher, not generated by a model

## Scientific Precision

- Use exact terminology from the source material. Do not simplify or paraphrase technical terms
- Preserve author names, theorem names, equation labels, and specific claims as they appear in the papers
- When summarising, distinguish between what the authors claim and what they demonstrate
- Cite specific sections, figures, or equations when referencing content from a document
- If uncertain about a detail, say so explicitly rather than guessing

## Working with Documents

- PDFs in this repository are annotated with highlights, margin notes, and inline scribbles from reading on a reMarkable tablet
- When extracting content, capture both the source text and any visible annotations
- Respect the topic-based folder structure when organising output
- File naming follows `YYYYMMDD_descriptive_title_pdf.pdf` -- the date is when the paper was added, not its publication date

## Output Format

- Prefer markdown for notes and summaries
- Use structured formats (tables, lists, headings) over prose when presenting extracted data
- Keep summaries concise but technically complete -- no padding, no redundant context-setting

## Knowledge Graph

This repository includes a nanograph property graph (`readings.nano/`) that models relationships between papers, authors, concepts, and manuscripts. The schema is in `readings.pg`, data in `seed.jsonl`, and canned queries in `readings.gq`.

When adding or enriching papers, update the graph:

1. Append nodes and edges to `seed.jsonl`
2. Reload: `nanograph load readings.nano --data seed.jsonl --mode merge`

See `.agents/skills/nanograph/SKILL.md` for the full JSONL format, CLI reference, slug conventions, and workflow details.
